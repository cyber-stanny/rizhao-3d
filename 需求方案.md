# 《日照 3D 街区漫游导览网页》最终需求文档与技术方案

版本：V1.0
项目范围：**零素材版本 / 程序化生成 3D 城市 / 不做真实街景 / 不依赖外部图片和模型**
项目目标：先做一个可以运行、可以展示、可以交互、可以漫游的 3D 日照城市街区 Demo。

---

# 一、项目结论

本项目第一版采用：

**React + Vite + Three.js / React Three Fiber**

实现一个网页端 3D 城市街区漫游系统。
第一版不需要你提前提供图片、3D 模型、全景图、航拍图、街景素材或真实地图数据。

所有城市元素先通过代码生成：

建筑群；
道路；
海面；
沙滩；
绿地；
广场；
港口；
灯塔；
地标标记；
路线连线；
景点卡片；
镜头飞行动画；
WASD 漫游。

第一版的定位是：

**低多边形风格的日照 3D 海滨城市街区漫游 Demo。**

不是：

真实街景；
真实城市数字孪生；
百度街景 / Google Street View 级别系统；
真实全景图漫游；
真实日照地图复刻。

# 二、需求文档 PRD

## 1. 项目名称

推荐名称：

**日照 3D 城市街区漫游导览系统**

备选名称：

**日照 3D 海滨城市导览网页**
**AI 驱动的日照城市数字名片**
**日照 3D 文旅漫游 Demo**
**日照城市 3D 互动展示平台**

---

## 2. 项目定位

本项目是一个基于网页的 3D 城市导览系统。用户打开网页后，可以进入一个虚拟生成的日照海滨城市街区，在 3D 场景中查看城市建筑、海岸线、道路、景点地标，并通过点击、镜头飞行、自动路线和第一人称漫游来浏览城市。

项目重点是：

做出一个有视觉冲击力的 3D 城市网页；
体现“日照海滨城市”的主题；
实现街区级 3D 漫游体验；
实现景点导览和路线展示；
不依赖外部素材，先让项目完整跑起来。

---

## 3. 核心目标

### 3.1 产品目标

完成一个网页端 3D 街区漫游 Demo，具备城市导览、景点展示、路线游览和基础漫游功能。

用户可以：

进入 3D 日照城市；
自由旋转、缩放、查看城市；
点击地标查看介绍；
镜头自动飞到目标地标；
选择路线自动游览；
使用键盘进入简单街区漫游。

---

### 3.2 技术目标

使用 Web 前端技术完成 3D 渲染和交互。

第一版要求：

无需外部 3D 模型；
无需真实图片；
无需真实地图服务；
全部核心视觉元素由代码生成；
支持浏览器直接访问；
代码结构清晰，后续可替换真实素材。

Three.js 官方文档提供相机、几何体、灯光、动画、Raycaster 等 3D 图形能力，适合网页端 3D 场景和交互实现；React Three Fiber 是 three.js 的 React 渲染器，适合把 3D 场景拆成可复用组件；Vite 官方定位是面向现代 Web 项目的构建工具，提供开发服务器和生产构建能力。([threejs.org][1])

---

### 3.3 展示目标

第一版最终效果应做到：

打开网页后，有一个日照主题的 3D 海滨城市；
能看到海面、沙滩、城市建筑、道路和地标；
能点击“万平口”“灯塔风景区”“东夷小镇”等点位；
能弹出景点卡片；
能自动镜头飞行；
能进行简单的 WASD 街区漫游；
整体适合比赛、课程设计、项目答辩和演示录屏。

## 1. 场景主题

场景主题为：

**日照海滨城市街区**

需要有明显的海滨城市特征：

海面；

海岸线；

港口区域；

---

## 2. 场景组成

| 元素   | 实现方式                 | 是否需要外部素材 |
| ---- | -------------------- | -------: |
| 城市地面 | PlaneGeometry        |      不需要 |
| 海面   | PlaneGeometry + 动画材质 |      不需要 |
| 沙滩   | 平面区域                 |      不需要 |
| 道路   | 长方形平面 / 线条           |      不需要 |
| 建筑群  | BoxGeometry 随机生成     |      不需要 |
| 绿地   | 平面 + 简单树模型           |      不需要 |
| 树木   | 圆柱体树干 + 球体树冠         |      不需要 |
| 灯塔   | 圆柱体 + 圆锥体 + 点光源      |      不需要 |
| 港口   | 平台 + 集装箱方块 + 船只概念模型  |      不需要 |
| 车站   | 几何体拼接建筑              |      不需要 |
| 地标标记 | 发光球体 + 文字标签          |      不需要 |
| 路线线条 | 曲线 / Line            |      不需要 |
| 天空背景 | 渐变背景 / 简单天空盒         |      不需要 |

---

## 3. 程序化城市生成规则

第一版城市不使用真实地图数据，而是通过规则生成。

### 3.1 城市布局

建议布局：

```text
东侧：海面
海岸线：沿城市右侧展开
中部：旅游街区、商业建筑、道路
西侧：车站、大学城、城市入口
南侧：港口、灯塔
北侧：森林公园、绿地
```

### 3.2 建筑生成规则

建筑以街区网格方式生成。

规则示例：

```text
按 8 x 6 街区生成建筑；
每个街区包含 3–8 栋建筑；
建筑高度随机；
靠近海岸的建筑更低；
城市中心建筑更高；
景点周围预留广场空间；
道路区域不生成建筑。
```

### 3.3 道路生成规则

使用横纵道路组成街区网格。

包括：

主干道；
支路；
海滨路；
景点连接路；
广场通道。

### 3.4 海岸线生成规则

第一版可以使用简化曲线：

```text
海面位于场景右侧；
沙滩位于城市和海面之间；
海岸线用曲线或不规则边界表现；
海面可加入轻微动态波动效果。
```

# 九、核心功能需求

## 1. 视角浏览功能

### 功能描述

用户可以用鼠标查看 3D 城市场景。

### 操作

| 操作       | 效果         |
| -------- | ---------- |
| 鼠标左键拖动   | 旋转视角       |
| 鼠标滚轮     | 缩放         |
| 鼠标右键拖动   | 平移，可选      |
| 点击“重置视角” | 回到默认城市全景   |
| 点击“俯瞰模式” | 切换到城市鸟瞰视角  |
| 点击“海岸视角” | 切换到海边低角度视角 |

---

## 2. 地标点击功能

### 功能描述

用户点击 3D 场景中的地标标记后，系统展示对应信息。

### 流程

```text
点击地标
→ 识别地标 ID
→ 更新选中状态
→ 镜头飞向地标
→ 打开右侧信息卡片
→ 高亮当前地标
```

### 信息卡片内容

| 字段   | 内容           |
| ---- | ------------ |
| 名称   | 万平口景区        |
| 类型   | 海滨景区         |
| 简介   | 一句话介绍        |
| 详细介绍 | 100–200 字    |
| 标签   | 海滨、拍照、亲子等    |
| 推荐时间 | 1–3 小时       |
| 操作按钮 | 飞到这里、加入路线、关闭 |

---

## 3. 镜头飞行功能

### 功能描述

点击地标或路线节点时，相机平滑移动到目标位置。

### 要求

飞行时间：1–2 秒；
飞行中镜头始终看向目标点；
支持中途打断；
到达后展示信息卡片；
自动游览时连续执行。

---

## 4. 自动游览功能

### 功能描述

用户点击“自动游览”后，系统按照预设路线依次展示地标。

### 流程

```text
点击自动游览
→ 选择默认路线或用户已选路线
→ 镜头飞向第一个地标
→ 展示地标介绍
→ 停留 3 秒
→ 飞向下一个地标
→ 所有地标完成后回到城市全景
```

### 第一版按钮

| 按钮     | 是否做 |
| ------ | --: |
| 开始自动游览 |   做 |
| 退出自动游览 |   做 |
| 暂停     |  可选 |
| 继续     |  可选 |
| 跳过当前景点 |  可选 |

---

## 5. 第一人称街区漫游功能

### 功能描述

用户进入漫游模式后，可以在 3D 城市中移动，模拟街区漫游。

### 操作

| 按键 / 操作 | 功能            |
| ------- | ------------- |
| W       | 前进            |
| S       | 后退            |
| A       | 左移            |
| D       | 右移            |

## 2. 镜头飞行

### 实现方式

用插值算法控制相机位置。

```ts
currentCameraPosition → targetCameraPosition
currentLookAt → targetLookAt
```

### 动画逻辑

```ts
function flyToSpot(spot) {
  start = camera.position;
  end = spot.cameraPosition;
  target = spot.position;

  animate({
    duration: 1500,
    onUpdate: (progress) => {
      camera.position.lerpVectors(start, end, ease(progress));
      camera.lookAt(target);
    }
  });
}
```

---

## 3. 自动游览

自动游览本质是多个镜头飞行动画串行执行。

```text
route.spotIds = ["wanpingkou", "lighthouse", "dongyi-town"]

for each spotId:
  flyToSpot(spotId)
  wait 3 seconds
  show spot info
```

状态：

```ts
tourStatus: "idle" | "playing" | "stopped";
currentTourIndex: number;
currentRouteId: string;
```

### 边界限制

```ts

```

| 地标不够像真实地点 | 没有真实模型      | 用图标、标签、文字介绍增强识别         |
| 漫游穿模      | 没有复杂碰撞      | 第一版限制范围，不做精细碰撞          |
| 移动端操作困难   | 手机端不适合 WASD | 移动端只保留浏览和点击             |
| 功能过多延期    | 范围失控        | 先完成 P0，再做 P1/P2         |
| 页面加载慢     | 几何体过多       | 共享几何体、减少材质、懒加载 UI       |
